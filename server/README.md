# Server

Express.js бэкенд с in-memory хранилищем на 1 000 000 элементов.

## Стек

- Express.js 5
- TypeScript
- In-memory хранилище (без БД)

## Запуск

```bash
npm install
npm run dev       # dev-режим с hot reload (tsx watch)
```

## Скрипты

| Команда | Описание |
|---------|----------|
| `npm run dev` | Запуск с hot reload через tsx |
| `npm run build` | Компиляция TS → JS в `dist/` |
| `npm start` | Запуск скомпилированного сервера |

## Архитектура

### Хранилище (`store.ts`)

Четыре структуры данных, работающие парами:

```
allItems[]  + allSet       — все элементы
selected[]  + selectedSet  — выбранные элементы
```

- **Массив** — для пагинации и сохранения порядка
- **Set** — для O(1) проверки существования (дедупликация, исключение выбранных)

При старте `allItems` заполняется числами от 1 до 1 000 000.

### API

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/api/items?page=1&filter=42` | Невыбранные элементы. Пагинация по 20, фильтр по подстроке ID |
| POST | `/api/items/batch` | Добавить новые ID. Дедупликация — существующие игнорируются |
| GET | `/api/selected?page=1&filter=42` | Выбранные в порядке D&D. Пагинация по 20 |
| POST | `/api/selected/batch` | Переместить элементы в выбранные |
| DELETE | `/api/selected/batch` | Убрать из выбранных |
| PUT | `/api/selected/reorder` | Drag & Drop — `{ itemId, beforeId }` |
| GET | `/api/health` | Health check |

### Особенности

- **Пагинация без полной фильтрации.** `getItems` итерирует массив и собирает ровно `offset + limit` элементов, не создавая промежуточный массив на 1М записей.
- **Дедупликация.** При добавлении новых элементов `allSet.has(id)` гарантирует уникальность за O(1).
- **Reorder.** Операция `{ itemId, beforeId }` — вытаскиваем элемент из текущей позиции и вставляем перед указанным. Если `beforeId: null` — в конец.
- **Раздача статики.** В продакшне Express отдаёт билд клиента из `../client/dist`.
